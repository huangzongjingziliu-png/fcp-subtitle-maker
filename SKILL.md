# FCP Subtitle Maker

将音频或视频文件直接转换为Final Cut Pro可导入的.fcpxml字幕文件。

## 核心功能

1. **严格使用用户文字稿** - 中文内容绝不使用语音识别结果
2. **智能气口切分** - 按音频停顿和标点自动切分长句
3. **单语字幕生成** - 生成中文或英文字幕
4. **双轨道双语字幕** - 中英文独立轨道，可在FCP中单独开关
5. **提取已有字幕翻译** - 从视频提取硬字幕/软字幕，翻译成英文

## 使用场景

### 场景1：生成单语字幕（中文或英文）
- 输入：视频/音频 + 文字稿（中文或英文）
- 输出：单语字幕文件（中文或英文轨道）

### 场景2：生成双语字幕
- 输入：视频/音频 + 中文文字稿 +（可选）英文翻译稿
- 输出：双轨道双语字幕（中英独立轨道）

### 场景3：根据带字幕的视频生成双语字幕
- 输入：带中文字幕的视频（硬字幕或软字幕）
- 处理：提取中文 → AI翻译成英文 → 生成双轨道双语字幕
- 输出：双语字幕（中英独立轨道）

**必需**：

**可选**：
- 中文文字稿（.txt / .docx / .pdf）
  - 提供文字稿 → 使用文字稿内容，更准确
  - 不提供文字稿 → 使用语音识别结果（准确率较低）

  - 提供英文稿 → 直接对齐使用
  - 不提供英文稿 → AI将根据中文精简翻译

1. 提取视频中文字幕（OCR或解封装）
2. AI翻译成英文

## 执行流程

### 1. 确认帧率
- 视频文件：自动识别，向用户确认
- 音频文件：询问用户，默认25fps

### 2. 语音识别获取精确时间轴（强制步骤）

**⚠️ 这是必须步骤，不可跳过！禁止简单按等分时长分配字幕时间。**

**目的**：获取每句话的精确起止时间，确保字幕与语音节奏完全同步。

**识别方法（按优先级）**：
1. **平台自带功能**（`audios_understand`）：可直接返回带时间戳的识别结果
2. **本地 whisper**：使用 openai-whisper 命令行工具
3. **在线 API**：提示用户安装或使用

**Whisper 命令示例**：
```bash
# 提取音频（如源文件是视频）
ffmpeg -i input.mp4 -vn -acodec pcm_s16le -ar 16000 -ac 1 output.wav

# 运行 whisper（获取逐句时间戳）
whisper output.wav --model medium --language zh --output_format json --output_dir ./whisper_output

# 或使用 SRT 格式（更直观）
whisper output.wav --model medium --language zh --output_format srt --output_dir ./whisper_output
```

**Whisper JSON 输出解析**：
```python
# whisper 输出的 segments 格式
# {"segments": [{"start": 0.0, "end": 2.5, "text": "识别文字"}, ...]}
import json
with open('whisper_output/output.json') as f:
    result = json.load(f)
segments = result['segments']
# 每个 segment 包含 start, end, text
```

**SRT 格式解析**：
```python
# SRT 格式示例：
# 1
# 00:00:15,000 --> 00:00:19,760
# 巍巍贺兰山横亘西北大地
def parse_srt(srt_path):
    with open(srt_path, 'r', encoding='utf-8') as f:
        content = f.read()
    segments = []
    blocks = content.strip().split('\n\n')
    for block in blocks:
        lines = block.strip().split('\n')
        if len(lines) >= 3:
            time_line = lines[1]
            start_str, end_str = time_line.split(' --> ')
            start = srt_time_to_seconds(start_str)
            end = srt_time_to_seconds(end_str)
            text = ' '.join(lines[2:])
            segments.append({'start': start, 'end': end, 'text': text})
    return segments
```

**关键原则**：
- **必须**使用语音识别获取每一句的精确 start/end 时间
- **禁止**按总时长等分分配时间（如 total_duration / segment_count）
- **禁止**使用固定间隔（如每3秒一句）
- 识别结果只用于**时间轴对齐**，**不用于字幕内容**
- 字幕内容严格使用用户提供的文字稿

### 3. 中文文字稿与时间轴对齐

**核心任务**：将用户文字稿切分后的内容，与语音识别的时间轴精确匹配。

**对齐方法**：

**方法A：基于文字稿切分 + Whisper segments 映射（推荐）**
1. 按标点和长度切分文字稿为段落
2. 将切分结果与 Whisper segments 逐一匹配
3. 匹配规则：
   - 逐个 Whisper segment 检查，累积文字直到覆盖一个完整切分段
   - 该切分段的 start = 第一个匹配 segment 的 start
   - 该切分段的 end = 最后一个匹配 segment 的 end
4. 输出带精确时间戳的字幕列表

**对齐示例**：
```
文字稿切分：
  段1: "巍巍贺兰山横亘西北大地"
  段2: "巍峨山体间"
  段3: "先民以石为笔"

Whisper segments：
  seg1: [0.0 - 3.2] "巍巍贺兰山横亘西北大地"  → 匹配段1
  seg2: [3.2 - 5.0] "巍峨山体间"              → 匹配段2
  seg3: [5.0 - 8.5] "先民以石为笔"            → 匹配段3

对齐结果：
  段1: start=0.0, end=3.2, text="巍巍贺兰山横亘西北大地"
  段2: start=3.2, end=5.0, text="巍峨山体间"
  段3: start=5.0, end=8.5, text="先民以石为笔"
```

**方法B：Whisper 输出 SRT 直接使用（无需文字稿时）**
- 当用户未提供文字稿时，直接使用 Whisper 的 SRT 输出作为时间轴
- 但仍然推荐用户提供文字稿，因为语音识别的中文准确率有限

**⚠️ 禁止做法**：
- ❌ `start = i * (total_duration / num_segments)` — 等分时间
- ❌ `duration = 3.0` — 固定每句3秒
- ❌ 忽略语音识别结果，凭感觉分配时间

**切分规则**：
| 标点 | 处理方式 | 说明 |
|------|----------|------|
| 句号（。） | 必切 | 句子结束 |
| 分号（；） | 必切 | 并列分句 |
| 逗号（，） | 可切 | 根据长度和气口判断 |
| 顿号（、） | 可切 | 长列表时切分 |
| 长句（>25字） | 强制切 | 确保可读性 |

**切分示例**：
```
原文：贺兰山岩画景区旅游资源丰富，5900余幅岩画和西夏、明代题刻、长城等历史遗迹竞相辉映

切分后：
1. 贺兰山岩画景区旅游资源丰富
2. 5900余幅岩画和西夏明代题刻
3. 长城等历史遗迹竞相辉映
```

**标点清理**（切分后执行）：
- 去除：，。、；：！？""''（）空格
- 保留：「」《》·—
- 转换：""→「」 ''→「」

### 4. 双语字幕生成（可选）

**用户提供英文稿时**：
- 读取英文文稿
- 段落数对齐：
  - 等于中文 → 一一对应
  - 少于中文 → **AI精简翻译补充**
  - 多于中文 → 适当合并

**未提供英文稿时**：
- AI逐段分析中文语义
- 精简翻译为英文
- 确保段数一致

**精简翻译原则**：
- 保留核心意思
- 删减修饰性从句
- 每段60-100字符
- 包含关键词（地名、数字）

**双语字幕格式 - 独立轨道**：
```xml
<!-- 中文轨道 - Lane 1（上方） -->
<title name="贺兰山岩画" lane="1">贺兰山岩画</title>

<!-- 英文轨道 - Lane 2（下方） -->
<title name="Helan Rock Art" lane="2">Helan Rock Art</title>
```

**轨道位置和字号说明**：
- **Lane 1** = 轨道1 = **中文** = 画面上方 = 字号 44 = Position `0 -300`
- **Lane 2** = 轨道2 = **英文** = 画面下方 = 字号 38 = Position `0 -420`
- FCP Position 坐标系：Y轴负值向下，绝对值越大越靠画面底部
- 中文在上（Y值绝对值小，靠近画面中上部），英文在下（Y值绝对值大，靠近画面底部）
- 英文比中文字号小，避免喧宾夺主

**优势**：
- 中文在上、英文在下，符合阅读习惯
- 轨道1和轨道2完全独立
- 在FCP中可单独开关、调整样式
- 可分别导出

**当用户只需要中文或英文字幕时**：

**处理流程**：
1. 读取用户提供的文字稿（中文或英文）
2. 按气口和标点智能切分
3. 通过语音识别获取时间轴
4. 生成单轨道字幕

**输出**：
- 单语字幕文件（仅一个轨道）

### 6. 已有字幕翻译（场景3）

**当用户已有中文字幕视频，需要翻译成英文时**：

**步骤1：提取中文字幕**

**硬字幕**（嵌入画面）：
1. 使用 OCR 工具提取字幕区域文字
2. 按时间轴整理字幕段

**软字幕**（内嵌字幕流）：
1. 解封装视频，提取字幕流（SRT/ASS格式）
2. 解析字幕文本和时间轴

**提取命令示例**：
```bash
# 提取软字幕（MP4容器）
ffmpeg -i input.mp4 -map 0:s:0 subtitles.srt

# 硬字幕需要OCR处理
# 使用帧提取+OCR识别字幕区域
ffmpeg -i input.mp4 -vf "fps=1,select='gt(scene,0.3)'" frames/%04d.png
```

**步骤2：AI翻译**
- 逐段分析中文语义
- 精简翻译为英文
- 保持时间轴不变

**步骤3：生成双轨道字幕**
- 中文：Lane 1（轨道1，上方，Position `0 -300`，字号44）
- 英文：Lane 2（轨道2，下方，Position `0 -420`，字号38）

### 7. 生成FCPXML

执行脚本：`generate_fcpxml.py`

**三种模式**：
- 单语模式：生成单个轨道字幕
- 双语模式：生成双轨道字幕（中英独立）
- 提取翻译模式：提取已有字幕并翻译

输出：`.fcpxml` 文件（可直接导入FCP）

## 注意事项

### 场景1（单语字幕）
1. **严格使用文字稿** - 语音识别只用于时间轴
2. **支持中文或英文** - 根据文字稿语言生成对应字幕
3. **单轨道输出** - 仅生成一个语言轨道

### 场景2（双语字幕）
1. **中文必须用文字稿** - 语音识别只用于时间轴
2. **必须使用语音识别获取时间轴** - 禁止等分时长、固定间隔
3. **切分会丢失部分标点** - 保留关键标点「」《》·—
4. **英文可能不匹配** - AI会自动精简翻译确保段数一致
5. **双轨道是独立的** - 可在FCP中分别控制
6. **中文在上、英文在下** - Lane 1 中文 position=`0 -300`，Lane 2 英文 position=`0 -420`

### 场景3（已有字幕翻译）
1. **硬字幕提取需要OCR** - 识别准确率依赖字幕清晰度
2. **时间轴保持不变** - 沿用原有字幕时间
3. **翻译基于已有字幕** - 不再进行气口切分
4. **建议提供原字幕文件** - 如有SRT/ASS文件，直接读取避免OCR误差
5. **中文在上、英文在下** - 同场景2位置设置
